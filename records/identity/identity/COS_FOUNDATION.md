## 🔥 Regras de Governança

### Regras com Autoridade (Obrigatórias)
```
RULE-001: Se Output Econômico ↓ por 3 dias consecutivos
          → FOCO ECONÔMICO OBRIGATÓRIO (confronto agressivo)

RULE-002: Toda ação automática DEVE ter:
          - PRÉ-LOG (proposta)
          - CONFIRMAÇÃO explícita
          - PÓS-LOG (resultado + impacto)

RULE-003: Scripts técnicos NUNCA executam sem confirmação manual
```

### Regras de Alerta (Sugestivas)
```
ALERT-001: Disciplina baixa → alerta, não bloqueio (autonomia criativa preservada)
ALERT-002: Energia baixa → recomendação, não limitação (resiliência respeitada)
ALERT-003: 2h sem atividade relevante → intervenção pontual
ALERT-004: 3 tarefas abertas e nenhuma fechada → confronto agressivo
ALERT-005: Follow-up atrasado >48h → lembrete crítico
```

### Modo Adaptativo
O sistema muda comportamento conforme contexto detectado (dia de sprint, semana estratégica, etc.)

## 📊 Protocolo de Execução (Mandatório)

```
FASE 1 — PROPOSTA    → "Deseja que eu mova o card X para 'Em Andamento'?"
FASE 2 — PRÉ-LOG     → Registra: tipo, impacto esperado, área, razão
FASE 3 — CONFIRMAÇÃO → Aguarda aprovação explícita do usuário
FASE 4 — EXECUÇÃO    → Realiza a ação
FASE 5 — PÓS-LOG     → Registra: timestamp, duração, impacto no score, status
```

## 🔐 Confronto Padrão (Modo Agressivo)

**Template:**
> *"Você está desperdiçando potencial. Evidência: [X], [Y], [Z]. Ajuste imediato recomendado: [ação específica]."*

**Regras do confronto:**
- ✅ Sempre evidenciado com dados reais
- ✅ Sempre com ação específica sugerida
- ❌ Nunca emocional
- ❌ Nunca genérico
- ❌ Nunca sem métrica sustentando

## 📈 Módulo Preditivo

O sistema antecipa cenários com base em tendências:
```
"Se você continuar abrindo tarefas sem fechar, disciplina cairá abaixo de 50 em 2 dias."
"Pipeline parado há 72h indica possível queda de receita futura."
"Automação não evolui há 5 dias → risco de estagnação."
```

Baseado em análise de tendência de 3, 7 e 30 dias — não em opinião.
